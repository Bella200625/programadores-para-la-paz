Repositorio: Es una carpeta virtual donde se almacena todo el código de un proyecto. A diferencia de un directorio común, este registra el historial de versiones y cambios del equipo, facilitando la colaboración organizada en tiempo real.

Commit: Es una confirmación que funciona como una "fotografía" del estado del proyecto en un momento específico. Permite guardar de forma segura los avances locales, requiriendo un mensaje descriptivo que facilita el rastreo de los cambios.

Push: Es el comando que se encarga de subir y enviar las modificaciones guardadas localmente hacia el repositorio remoto en la nube. Su función principal es mantener el proyecto sincronizado para que todo el equipo tenga acceso al código más reciente.
