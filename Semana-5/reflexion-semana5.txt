Las APIs son herramientas esenciales para que las comunidades organicen su información, ya que permiten pasar de solo leer datos (con métodos GET) a enviar reportes activos mediante métodos POST. Aunque en la práctica de esta semana nos enfocamos en la lectura y vimos la estructura de POST y JSON de forma teórica, queda claro que este canal es el que permite al ciudadano mandar paquetes de información específicos, como alertas de infraestructura o seguridad. Al usar un formato estándar como JSON, la API recibe, procesa y ordena cada reporte de forma automática en el servidor, evitando el caos de los mensajes sueltos o las denuncias desorganizadas. Para nosotros como desarrolladores, entender este flujo cliente-servidor es clave, pues nos enseña cómo la tecnología puede centralizar las necesidades de un barrio, transformando un mensaje enviado por un usuario en una respuesta o solución guardada y gestionada eficientemente para el beneficio de todos.
