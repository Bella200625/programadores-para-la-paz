Durante la práctica comprobé que el servidor estaba funcionando correctamente al acceder a la ruta /registro desde el navegador mediante una ruta GET de prueba. Al principio intenté acceder directamente a la ruta POST desde el navegador y apareció el mensaje "Cannot GET /registro", lo que me permitió entender que las rutas POST no pueden probarse de esa forma. También intenté realizar la prueba desde la terminal utilizando curl, pero encontré dificultades con la forma en que PowerShell interpretaba el comando. Aun así, comprendí que la ruta POST está diseñada para recibir datos en formato JSON mediante req.body y devolver una respuesta en formato JSON con la información recibida.
